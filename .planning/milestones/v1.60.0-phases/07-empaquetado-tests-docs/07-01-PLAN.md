Contenido requerido (ajusta solo comentarios, NO los valores load-bearing):

```dockerfile
# Imagen base: Playwright pineado a la versión de playwright en uv.lock (1.60.0).
# Mismatch de tag = browsers no encontrados en runtime.
FROM mcr.microsoft.com/playwright/python:v1.60.0-noble

# Deps de sistema: FFmpeg (montaje de vídeo) + Poppler (fallback pdf2image en ingesta).
RUN apt-get update \
    && apt-get install -y --no-install-recommends ffmpeg poppler-utils \
    && rm -rf /var/lib/apt/lists/*

# Binario uv para instalación reproducible de dependencias.
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv

WORKDIR /app

# Copiar manifiestos primero para cachear la capa de dependencias.
COPY pyproject.toml uv.lock ./

# Instalación reproducible desde el lock; sin dev deps y sin el extra 'record'
# (torch/whisperx ~2GB son solo para modo record — opcional, ver README).
RUN uv sync --frozen --no-dev

# Copiar el código fuente del proyecto.
COPY src ./src

# Asegurar que el Chromium pineado del paquete está disponible
# (la imagen base ya lo trae; este paso garantiza coincidencia con playwright 1.60.0).
RUN uv run playwright install chromium

# Las claves de API se inyectan en runtime vía variables de entorno o .env montado:
#   docker run -e ANTHROPIC_API_KEY=... -e ELEVENLABS_API_KEY=... ...
# NUNCA se hornean en la imagen (ver threat_model T-07-02).

ENTRYPOINT ["uv", "run", "avideo"]
CMD ["--help"]
```

Reglas duras:
- El tag base DEBE ser `v1.60.0-noble` (coincide con playwright en uv.lock). No cambiar.
- `uv sync` DEBE llevar `--frozen` (reproducible, falla si lock desactualizado) y `--no-dev`. NO añadir `--extra record`.
- ENTRYPOINT = `["uv", "run", "avideo"]` para que `docker run <img> generate --bullets ... --duration ...` funcione.
- NO instalar torch ni whisperx en la imagen por defecto.
- NO copiar `.env`, `workdir/`, ni secretos al build.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build context → imagen Docker | Archivos del repo que entran en la imagen (riesgo: hornear secretos/.env) |
| registry externo → imagen base | `mcr.microsoft.com/...` y `ghcr.io/astral-sh/uv:latest` son dependencias de supply-chain |
| runtime env → contenedor | API keys (ANTHROPIC/ELEVENLABS) inyectadas en runtime, no en build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Imagen base / binario uv (supply-chain) | mitigate | Pinear tag de Playwright a `v1.60.0-noble` (no `latest`); `uv sync --frozen` ata las deps a hashes de uv.lock. `ghcr.io/astral-sh/uv:latest` se acepta como riesgo menor documentado (binario estático, herramienta de build). |
| T-07-02 | Information Disclosure | Secretos (ANTHROPIC_API_KEY, ELEVENLABS_API_KEY) | mitigate | NUNCA copiar `.env` ni hornear claves en la imagen; `.dockerignore` excluye `.env`; claves se inyectan en runtime con `-e` o `--env-file`. Comentario explícito en el Dockerfile. |
| T-07-03 | Tampering | torch/whisperx en imagen por defecto | accept | Excluidos de la imagen por defecto (extra `record` opcional). Sin superficie adicional en la imagen `auto`. |
