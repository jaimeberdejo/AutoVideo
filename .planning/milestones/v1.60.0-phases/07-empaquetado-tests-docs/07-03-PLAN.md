Comando: `avideo generate`
- `--bullets PATH` (requerido) — ruta a bullets.yaml
- `--duration INT` (requerido, min 1) — duración objetivo en segundos
- `--voice {elevenlabs|record}` (opcional, default elevenlabs)
- `--slides-mode {auto|hybrid|manual}` (opcional, default auto)
- `--level INT` (opcional, 1-4, default 4) — 1 pausa cada etapa, 4 totalmente autónomo
- `--context PATH` (opcional) — documento de contexto .pptx/.pdf/.md
- `--dry-run` (flag) — estima coste/tokens sin generar audio/vídeo
- `--burn-subs` (flag) — quema subtítulos en el vídeo
- `--verbose` (flag) — logging debug con tracebacks Rich

Formatos de config (leídos en planning):
- `config.yaml`: voice, slides_mode, level, wpm, voice_id, language (CLI > YAML > default)
- `bullets.yaml`: `title:` + `bullets:` (lista de strings)
- `theme.yaml`: tema visual (paleta/tipografías/espaciado) — existe en la raíz
- Env vars: `ANTHROPIC_API_KEY`, `ELEVENLABS_API_KEY` (vía .env / python-dotenv en dev)

Stack de instalación (CLAUDE.md):
- `uv sync` (deps por defecto, modo auto + elevenlabs)
- `uv run playwright install chromium` (NECESARIO tras instalar el paquete)
- Sistema: FFmpeg (montaje), Poppler (`poppler-utils`, para pdf2image) — en macOS `brew install ffmpeg poppler`, en Debian/Ubuntu `apt-get install ffmpeg poppler-utils`
- Modo record (opcional, pesado): torch==2.5.1 + torchaudio==2.5.1 ANTES de whisperx, luego `uv sync --extra record`
