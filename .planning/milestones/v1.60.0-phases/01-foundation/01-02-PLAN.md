<config_merge_note>
RunConfig already implements precedence in settings_customise_sources: (init_settings, YamlConfigSettingsSource, env_settings). The CLI must pass ONLY the values the user actually supplied as init kwargs so config.yaml fills the rest. Typer always passes a value for each parameter (its default if not given), which would clobber config.yaml. To preserve precedence, pass `None` as the Typer default for every OPTIONAL flag and build the RunConfig kwargs dict by INCLUDING ONLY keys whose value is not None (i.e. flags the user actually set). `bullets` and `duration` are required and always passed.
</config_merge_note>
