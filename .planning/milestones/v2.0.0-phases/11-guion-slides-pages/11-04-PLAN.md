    IMPLEMENTATION STRUCTURE:

    1. MODE SELECTION (SLD-01)
       Persist mode choice in session_state so it survives reruns:
         if "sld_mode" not in st.session_state:
             st.session_state["sld_mode"] = "auto"

       mode = st.radio(
           "Modo de diapositivas",
           ["Generar (auto)", "Subir las mías"],
           key="sld_mode_radio",
           horizontal=True,
       )
       slides_mode = "auto" if mode == "Generar (auto)" else "manual"
       st.session_state["sld_mode"] = slides_mode

       # Persist mode into run_config (for SlidesDispatchStage routing)
       rc = st.session_state.get("run_config", {})
       rc["slides_mode"] = slides_mode
       st.session_state["run_config"] = rc

    2. BUILD CONFIG (used in both paths)
       from avideo.models.config import RunConfig
       rc_kwargs = dict(st.session_state.get("run_config", {}))
       if "bullets" not in rc_kwargs:
           rc_kwargs["bullets"] = workdir.root / "bullets.yaml"
       config = RunConfig(**rc_kwargs)

    3a. AUTO PATH (SLD-02)
        slides_done = workdir.is_done("slides")
        verify_done = workdir.is_done("verify")

        If not slides_done: auto-launch slide generation
          from avideo.stages.slides_dispatch import SlidesDispatchStage
          from avideo.ui.bridge import run_stage
          run_stage(SlidesDispatchStage(), workdir, config)

        If slides_done and not verify_done: auto-launch verify
          from avideo.stages.verify_slides import VerifyStage
          run_stage(VerifyStage(), workdir, config)

        Progress display with @st.fragment(run_every="2s"):
          @st.fragment(run_every="2s")
          def _poll_slides_auto():
              from avideo.ui.bridge import stage_status, RunStatus, get_error
              s_slides = stage_status("slides", workdir)
              s_verify = stage_status("verify", workdir)
              if s_slides == RunStatus.ERROR:
                  st.error(f"Error generando slides: {get_error('slides')}")
              elif s_slides == RunStatus.RUNNING:
                  st.info("⏳ Generando diapositivas...")
              elif s_slides == RunStatus.DONE and s_verify == RunStatus.RUNNING:
                  st.info("⏳ Verificando calidad...")
              elif s_verify == RunStatus.DONE:
                  st.rerun()  # exit fragment loop; show thumbnails
          if not verify_done:
              _poll_slides_auto()
              return False

        Show thumbnails + badges once done:
          slides_out: SlidesOutput = workdir.read_checkpoint("slides", SlidesOutput)
          report: VerificationReport = workdir.read_checkpoint("verification", VerificationReport)
          verdict_by_idx = {v.slide_index: v for v in report.slides}

          cols_per_row = 3
          for row_start in range(0, len(slides_out.png_paths), cols_per_row):
              cols = st.columns(cols_per_row)
              for col_i, png_path in enumerate(slides_out.png_paths[row_start:row_start+cols_per_row]):
                  slide_idx = row_start + col_i
                  verdict = verdict_by_idx.get(slide_idx)
                  badge = badge_for_verdict(verdict) if verdict else "○"
                  with cols[col_i]:
                      st.image(png_path, caption=f"Slide {slide_idx + 1} {badge}", use_container_width=True)
                      if verdict and verdict.status != "ok":
                          with st.expander(f"Detalles QC slide {slide_idx + 1}"):
                              for issue in verdict.issues:
                                  st.write(f"• {issue}")
                              for sug in verdict.suggestions:
                                  st.write(f"→ {sug}")

        Variation button (SLD-02):
          if st.button("🔄 Pedir variación de slides", key="btn_slides_variation"):
              from avideo.ui.pipeline_ops import rerun_slides  # noqa: PLC0415
              workdir.invalidate_downstream("slides")  # also clears verify
              rerun_slides(workdir, config)
              st.rerun()

    3b. UPLOAD PATH (SLD-03)
        Read storyboard to know how many slides are expected:
          sb: StoryboardOutput = workdir.read_checkpoint("storyboard", StoryboardOutput)
          n_slides = len(sb.slides)

        For each slide index, show uploader + current status:
          all_uploaded = True
          for idx in range(n_slides):
              expected_name = f"slide_{idx:02d}.png"
              upload_path = workdir.root / "slides_user" / expected_name
              col_label, col_upload = st.columns([1, 3])
              with col_label:
                  st.write(f"Slide {idx + 1}")
              with col_upload:
                  uploaded = st.file_uploader(
                      f"Subir slide {idx + 1} (PNG/PDF)",
                      key=f"upload_{idx}",
                      type=["png", "pdf"],
                      label_visibility="collapsed",
                  )
                  if uploaded is not None:
                      from avideo.ui.pipeline_ops import write_uploaded_slide  # noqa: PLC0415
                      write_uploaded_slide(workdir, expected_name, uploaded.read())
                      st.success(f"slide_{idx + 1} subida.")
                  elif not upload_path.exists():
                      all_uploaded = False
                      st.info("Sin subir")
                  else:
                      st.success("Ya subida")
                      st.image(str(upload_path), width=200)

        "Verificar con Claude Vision" button (only once all slides present):
          verify_done = workdir.is_done("verify")
          if all_uploaded and not verify_done:
              if st.button("🔍 Verificar diapositivas (Claude Vision)", key="btn_verify"):
                  # Invalidate old verify done-marker (idempotent via bridge)
                  workdir.invalidate_downstream("slides")
                  # Run slides (manual ingest) then verify
                  from avideo.stages.slides_dispatch import SlidesDispatchStage
                  from avideo.stages.verify_slides import VerifyStage
                  from avideo.ui.bridge import run_stage
                  run_stage(SlidesDispatchStage(), workdir, config)
                  run_stage(VerifyStage(), workdir, config)
                  st.rerun()

          # Poll if running
          @st.fragment(run_every="2s")
          def _poll_slides_verify():
              from avideo.ui.bridge import stage_status, RunStatus, get_error
              sv = stage_status("verify", workdir)
              if sv == RunStatus.RUNNING or stage_status("slides", workdir) == RunStatus.RUNNING:
                  st.info("⏳ Verificando con Claude Vision...")
              elif sv == RunStatus.DONE:
                  st.rerun()
              elif sv == RunStatus.ERROR:
                  st.error(f"Error verificando: {get_error('verify')}")
          if all_uploaded and not verify_done:
              _poll_slides_verify()

          # Show verification report if done
          if verify_done:
              report: VerificationReport = workdir.read_checkpoint("verification", VerificationReport)
              verdict_by_idx = {v.slide_index: v for v in report.slides}
              for idx in range(n_slides):
                  upload_path = workdir.root / "slides_user" / f"slide_{idx:02d}.png"
                  verdict = verdict_by_idx.get(idx)
                  badge = badge_for_verdict(verdict) if verdict else "○"
                  col_img, col_report = st.columns([1, 2])
                  with col_img:
                      if upload_path.exists():
                          st.image(str(upload_path), caption=f"Slide {idx + 1} {badge}", width=200)
                  with col_report:
                      if verdict:
                          if verdict.status == "ok":
                              st.success(f"✅ Slide {idx + 1}: OK")
                          elif verdict.status == "warning":
                              st.warning(f"⚠️ Slide {idx + 1}: Advertencia")
                              for issue in verdict.issues:
                                  st.write(f"• {issue}")
                          else:
                              st.error(f"❌ Slide {idx + 1}: Problema")
                              for issue in verdict.issues:
                                  st.write(f"• {issue}")
                              for sug in verdict.suggestions:
                                  st.write(f"→ {sug}")

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser file upload → write_uploaded_slide → slides_user/ | Untrusted file bytes and filename from user's browser |
| slides_user/ PNG → VerifyStage → Anthropic API (base64) | User-supplied image sent to Claude Vision |
| storyboard bullets → verification prompt | User-authored content used in Claude Vision prompt (framed as untrusted reference in VerifyStage, T-06-01 — no change needed here) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04-01 | Tampering | st.file_uploader filename → slides_user/ | mitigate | Filename is overwritten with expected_name (slide_00.png, slide_01.png...) — user's original filename is ignored; write_uploaded_slide guards against traversal in the general case |
| T-11-04-02 | Tampering | uploaded file content → base64 → Anthropic API | accept | Image is passed to VerifyStage which enforces MAX_BYTES guard (T-06-02 from Phase 6); responsibility delegated to existing stage |
| T-11-04-03 | Denial of Service | large upload file | accept | Streamlit's default upload limit applies; pipeline_ops.write_uploaded_slide does not add a second size check for v2.0.0 |
| T-11-04-04 | Information Disclosure | ANTHROPIC_API_KEY | accept | Loaded from .env via load_dotenv() in app.py; never exposed in Streamlit UI |
| T-11-04-05 | Elevation of Privilege | slides_user/ path outside workdir | mitigate | write_uploaded_slide called with fixed expected_name (slide_XX.png), not raw filename — no traversal possible; additionally write_uploaded_slide raises ValueError on "/" in name |
</threat_model>
