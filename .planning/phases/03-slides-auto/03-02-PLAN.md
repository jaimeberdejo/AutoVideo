<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storyboard.json (LLM output) → render context | Model-generated slide text flows into Jinja2/HTML |
| theme generation prompt (storyboard summary) → Claude | Storyboard text summarized into the theme system/user prompt |
| runtime → external network | Render + dry-run must stay offline (D-08, CLI-06) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering (prompt injection via storyboard text into theme prompt) | resolve_theme call_structured | mitigate | Theme tool-use is forced + schema-constrained to ThemeConfig (only colors/fonts/spacing); summarize storyboard as untrusted reference, never as instructions. Worst case = odd colors, not arbitrary action |
| T-03-06 | Information disclosure | --dry-run / cost_estimator | mitigate | estimate_theme_tokens is pure arithmetic — no network, no API key read (preserves the offline dry-run contract; mirrors existing estimators) |
| T-03-07 | Denial of service | theme.yaml regeneration on every run | mitigate | Idempotency (D-03): existing theme.yaml is loaded, never regenerated — bounds API calls to once per project, and once written never re-billed |
| T-03-08 | Tampering | unknown/legacy visual_type → KeyError crash | mitigate | base template dispatch falls back to bullets_slide for unknown values (Pitfall 5); render never crashes on stale storyboard.json |
</threat_model>
