Playwright render pattern (VERIFIED end-to-end in RESEARCH, playwright 1.60.0, Chromium-1217):
```python
from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch()              # headless default; ONE per run
    page = browser.new_page(viewport={"width":1920,"height":1080}, device_scale_factor=1)
    page.set_content(html, wait_until="load")  # set_content has NO base URL → embed fonts base64
    page.evaluate("""async () => {
        await Promise.all([...document.fonts].map(f => f.load().catch(()=>{})));
        await document.fonts.ready;            # CRITICAL: .load() each face BEFORE .ready
    }""")
    page.screenshot(path=out, type="png", animations="disabled")  # full_page stays False
    browser.close()
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storyboard.json (LLM output) → Jinja2 template | Slide title/bullets are model-generated text rendered into HTML |
| theme.yaml (AI/user) → CSS injection | Theme strings (colors, font names) injected into `<style>` |
| runtime → external network | Render MUST NOT fetch any external resource (offline-only constraint D-08) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information disclosure / Tampering | integrations/playwright.py render | mitigate | `set_content` (no base URL) + base64 @font-face only; no `goto(http)`, no `<img src=http>`; offline render verified — nothing leaves the machine at render time |
| T-03-02 | Tampering (template/HTML injection) | macros.html.j2 rendering slide.title/bullets | mitigate | Rely on Jinja2 autoescape for ALL user/LLM text; only the trusted `icon()` SVG (generated by python-lucide, our code) is marked `|safe`. Bullet/title text is never `|safe` |
| T-03-03 | Tampering | theme color/font-name strings → `<style>` | accept | ThemeConfig fields are short strings from our own AI tool-call/user file; worst case is malformed CSS (broken render, not exfiltration). No JS context. Low risk; validated as str by Pydantic |
| T-03-04 | Denial of service | content overflow > 1080px (Pitfall 4) | mitigate | base template `overflow:hidden` + fixed 1920×1080 + DSF=1 → screenshot can never exceed/scroll; deterministic dimensions |
</threat_model>
