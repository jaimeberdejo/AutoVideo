- Largest-remainder (RESEARCH Pattern 3, D-06): `apportion_seconds(weights, total)` floors raw shares, distributes the integer remainder to the largest fractional parts → `sum == total` guaranteed. Emit `float(int_seconds)` to satisfy the `seconds: float` contract while keeping exact integer sums.
- Clamps (D-05, Pitfall 3): clamping seconds AFTER apportionment breaks the exact-sum invariant. Recommended order — clamp the WEIGHTS pre-apportionment, OR re-balance slack after clamping. Pick ONE and unit-test `sum(seconds) == target` with clamps active. RESEARCH recommends starting constants min≈8s, max≈45s; document chosen values in the SUMMARY.
- Weights (D-05): per-slide weight = f(#bullets, char length of title+bullets). Deterministic, no network, no LLM (D-08).
- word_budget (D-07): `round(seconds * wpm / 60)`; test for wpm ∈ {120,150,180}.
- Scriptwriter (D-09/D-10/D-11, Pattern 4): whole-script single call with per-slide budgets explicit in the prompt; `_max_drift(script, budgets)` = max over slides of `abs(len(narration.split()) - budget)/budget`; if > 0.25 → ONE re-call with a correction prompt; accept whatever returns even if still off (NO loop). Spanish (config.language), natural spoken tone. Forced tool-use via call_structured. The scriptwriter reads BOTH storyboard.json (content/coherence) and timings.json (per-slide budgets) — RESEARCH Open Question 4.
- max_tokens (Pitfall 7): scale generously for the whole-script call (e.g. 8192+, or scale by total word budget) to avoid truncation; Sonnet 4.6 ceiling is 64k.
- cost_estimator (D-15, RESEARCH Code Example): offline heuristic — `estimate_storyboard_tokens(num_bullets, duration)` etc. Use `load_bullets(config.bullets)` for the real bullet count. Sonnet 4.6 pricing $3/MTok in, $15/MTok out. NEVER call count_tokens (network) — anti-pattern that breaks the offline dry-run contract.
- Mock point (TEST-01): patch `avideo.stages.scriptwriter.call_structured`. Use 2 side_effects to drive the calibration-retry test (first off-budget, second still-off → assert exactly 2 calls, no third).
