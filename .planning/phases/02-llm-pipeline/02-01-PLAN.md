<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user filesystem → ingestion | `--context` file content is untrusted reference material parsed by fitz/python-pptx |
| user filesystem → bullets loader | `bullets.yaml` is user-controlled YAML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `config.context` suffix | mitigate | Validate suffix against `_DISPATCH` allow-list before opening; Typer already enforces exists/file-only (no traversal) |
| T-02-02 | Denial of Service | oversized/zip-bomb .pptx or huge PDF | mitigate | Truncate extracted text to `CONTEXT_TOKEN_CAP` (D-04); extraction reads text only (no rasterization) |
| T-02-03 | Tampering (side effect) | python-pptx `notes_slide` | mitigate | Guard `if slide.has_notes_slide` before access (Pitfall 6 — reading otherwise mutates the deck) |
| T-02-04 | Information Disclosure | logging extracted document text | mitigate | Never log raw extracted content at INFO; log only suffix + char count |
| T-02-05 | Tampering | malformed `bullets.yaml` | accept | `yaml.safe_load` (no arbitrary object construction) + Pydantic validation surfaces a clear error; single-user local tool |
</threat_model>
