## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local process → api.anthropic.com | outbound HTTPS LLM call; API key is the only credential |
| context document content → LLM prompt | untrusted reference text injected into the storyboard prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | prompt injection via context text | mitigate | System prompt frames context as untrusted reference material, not instructions; low impact (single-user, no downstream privilege) |
| T-02-07 | Information Disclosure | ANTHROPIC_API_KEY in logs/checkpoints/tracebacks | mitigate | Key read by SDK from env only; never placed in RunConfig or any model; do not echo it in Rich error messages; lazy client so it is never required to import |
| T-02-08 | Denial of Service | unbounded context inflating the prompt | mitigate | Context already truncated to CONTEXT_TOKEN_CAP in 02-01; storyboard max_tokens bounded |
| T-02-09 | Tampering | malformed tool_use output | mitigate | Pydantic `model_validate(block.input)` rejects non-conformant output; missing tool_use block → clear RuntimeError (surfaced by orchestrator as a Rich error, not a traceback) |
</threat_model>
